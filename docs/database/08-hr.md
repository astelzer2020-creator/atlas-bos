# HR Database Schema

## Overview

The **Workforce** bounded context manages employees, organizational structure, employment contracts, time-off policies, payroll integration references, and org chart visualization. All tables reside in PostgreSQL `workforce` schema with `organization_id`, RLS, soft delete, and audit columns.

| Entity | Table | Aggregate Root |
|--------|-------|----------------|
| Employee | `workforce.employees` | **Employee** |
| Department | `workforce.departments` | **Department** |
| Position | `workforce.positions` | **Position** |
| Employment Contract | `workforce.employment_contracts` | **EmploymentContract** |
| Time Off Policy | `workforce.time_off_policies` | **TimeOffPolicy** |
| Time Off Request | `workforce.time_off_requests` | **TimeOffRequest** |
| Payroll Run | `workforce.payroll_runs` | **PayrollRun** (integration ref) |
| Org Chart Node | `workforce.org_chart_nodes` | **OrgChart** (via nodes) |

```
├─────────────────────────────────────────────────────────────────────────┤

│    ├── employmentContracts[] (temporal employment terms)                │
│    ├── timeOffRequests[] (reference)                                    │
│    └── orgChartNodes[] (position in hierarchy)                          │
├─────────────────────────────────────────────────────────────────────────┤
│  Department (AR) — workforce.departments                                 │
│    ├── positions[] (reference)                                          │
│    └── self-referential parent_department_id                            │
├─────────────────────────────────────────────────────────────────────────┤
│  Position (AR) — workforce.positions                                     │
│    └── headcount, salary band, department link                          │
├─────────────────────────────────────────────────────────────────────────┤
│  EmploymentContract (AR) — workforce.employment_contracts                │
│    └── terms, compensation, dates (temporal validity)                   │
├─────────────────────────────────────────────────────────────────────────┤
│  TimeOffPolicy (AR) — workforce.time_off_policies                        │
│    └── accrual rules, entitlement configuration                         │
├─────────────────────────────────────────────────────────────────────────┤
│  TimeOffRequest (AR) — workforce.time_off_requests                       │
│    └── approval workflow state machine                                  │
├─────────────────────────────────────────────────────────────────────────┤
│  PayrollRun (AR) — workforce.payroll_runs                                │
│    └── external provider refs (Gusto, ADP); no local payroll calc       │
├─────────────────────────────────────────────────────────────────────────┤
│  OrgChart (AR) — workforce.org_chart_nodes                               │
│    └── reporting structure nodes (employee ↔ manager)                 │
└─────────────────────────────────────────────────────────────────────────┘
```

### Invariants

| Aggregate | Rule |
|-----------|------|
| **Employee** | `employee_number` unique per org; links to `atlas_core.users` when portal access enabled |
| **Department** | `code` unique per org; no circular parent references |
| **Position** | `code` unique per org; `filled_count ≤ headcount` |
| **EmploymentContract** | No overlapping active contracts per employee; `end_date >= start_date` |
| **TimeOffPolicy** | `accrual_rate >= 0`; policy type: `pto`, `sick`, `parental`, `unpaid`, `other` |
| **TimeOffRequest** | Status: `pending → approved \| rejected \| cancelled`; cannot exceed available balance |
| **PayrollRun** | Status: `draft → submitted → processing → completed \| failed`; immutable after `completed` |
| **OrgChartNode** | One active node per employee; manager cannot be self or descendant |

## Tables

### 1. `workforce.departments`

```sql
CREATE TABLE workforce.departments (
    id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    organization_id     UUID NOT NULL REFERENCES atlas_core.organizations(id),
    parent_department_id UUID REFERENCES workforce.departments(id),
    code                TEXT NOT NULL,
    name                TEXT NOT NULL,
    description         TEXT,
    cost_center_code    TEXT,
    manager_id          UUID REFERENCES workforce.employees(id),
    is_active           BOOLEAN NOT NULL DEFAULT true,
    metadata            JSONB NOT NULL DEFAULT '{}',
    created_at          TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at          TIMESTAMPTZ NOT NULL DEFAULT now(),
    deleted_at          TIMESTAMPTZ,
    created_by          UUID REFERENCES atlas_core.users(id),
    updated_by          UUID REFERENCES atlas_core.users(id),
    version             INTEGER NOT NULL DEFAULT 1
);

CREATE UNIQUE INDEX uq_departments_org_code_active
    ON workforce.departments (organization_id, lower(code)) WHERE deleted_at IS NULL;

CREATE INDEX idx_departments_organization_id ON workforce.departments (organization_id);
CREATE INDEX idx_departments_org_parent ON workforce.departments (organization_id, parent_department_id) WHERE deleted_at IS NULL;
```

```sql
    filled_count        INTEGER NOT NULL DEFAULT 0 CHECK (filled_count >= 0),
```

### 3. `workforce.employees`

```sql
CREATE TABLE workforce.employees (
    id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    organization_id     UUID NOT NULL REFERENCES atlas_core.organizations(id),
    user_id             UUID REFERENCES atlas_core.users(id),
    employee_number     TEXT NOT NULL,
    external_id         TEXT,

    last_name           TEXT NOT NULL,

    department_id       UUID REFERENCES workforce.departments(id),
    position_id         UUID REFERENCES workforce.positions(id),
    manager_id          UUID REFERENCES workforce.employees(id),
    hire_date           DATE NOT NULL,
    termination_date    DATE,
    employment_status   TEXT NOT NULL DEFAULT 'active'
                        CHECK (employment_status IN ('active', 'on_leave', 'terminated', 'suspended')),
    work_location       TEXT,
    timezone            TEXT NOT NULL DEFAULT 'UTC',
    avatar_url          TEXT,
    date_of_birth       DATE,
    gender              TEXT,
    emergency_contact   JSONB NOT NULL DEFAULT '{}',
    address             JSONB NOT NULL DEFAULT '{}',
    metadata            JSONB NOT NULL DEFAULT '{}',

    updated_at          TIMESTAMPTZ NOT NULL DEFAULT now(),
    deleted_at          TIMESTAMPTZ,
    created_by          UUID REFERENCES atlas_core.users(id),
    updated_by          UUID REFERENCES atlas_core.users(id),
    version             INTEGER NOT NULL DEFAULT 1,

    CONSTRAINT chk_employees_termination_after_hire CHECK (
        termination_date IS NULL OR termination_date >= hire_date
    )
);

CREATE UNIQUE INDEX uq_employees_org_number_active
    ON workforce.employees (organization_id, employee_number) WHERE deleted_at IS NULL;

CREATE UNIQUE INDEX uq_employees_org_work_email_active
    ON workforce.employees (organization_id, work_email) WHERE deleted_at IS NULL;

    ON workforce.employees (organization_id, user_id) WHERE user_id IS NOT NULL AND deleted_at IS NULL;

CREATE INDEX idx_employees_org_department ON workforce.employees (organization_id, department_id) WHERE deleted_at IS NULL;
CREATE INDEX idx_employees_org_manager ON workforce.employees (organization_id, manager_id) WHERE deleted_at IS NULL;
CREATE INDEX idx_employees_org_status ON workforce.employees (organization_id, employment_status) WHERE deleted_at IS NULL;
```

### 4. `workforce.employment_contracts`

```sql
CREATE TABLE workforce.employment_contracts (

    currency_code       CHAR(3) NOT NULL DEFAULT 'USD',
    working_hours       NUMERIC(5, 2) NOT NULL DEFAULT 40,
    notice_period_days  INTEGER NOT NULL DEFAULT 30,
    status              TEXT NOT NULL DEFAULT 'active'
                        CHECK (status IN ('draft', 'active', 'expired', 'terminated')),
    terms               JSONB NOT NULL DEFAULT '{}',
    signed_at           TIMESTAMPTZ,
    document_url        TEXT,
    metadata            JSONB NOT NULL DEFAULT '{}',
    created_at          TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at          TIMESTAMPTZ NOT NULL DEFAULT now(),
    deleted_at          TIMESTAMPTZ,
    created_by          UUID REFERENCES atlas_core.users(id),
    updated_by          UUID REFERENCES atlas_core.users(id),
    version             INTEGER NOT NULL DEFAULT 1,

    CONSTRAINT chk_contracts_end_after_start CHECK (end_date IS NULL OR end_date >= start_date)
);

CREATE UNIQUE INDEX uq_employment_contracts_org_number
    ON workforce.employment_contracts (organization_id, contract_number) WHERE deleted_at IS NULL;

CREATE INDEX idx_employment_contracts_org_employee
    ON workforce.employment_contracts (organization_id, employee_id) WHERE deleted_at IS NULL;

CREATE INDEX idx_employment_contracts_org_active
    ON workforce.employment_contracts (organization_id, employee_id, status)
    WHERE deleted_at IS NULL AND status = 'active';
```

## Row-Level Security (RLS)

```sql
DO $$
DECLARE tbl TEXT;
BEGIN
    FOR tbl IN SELECT tablename FROM pg_tables WHERE schemaname = 'workforce' LOOP
        EXECUTE format('ALTER TABLE workforce.%I ENABLE ROW LEVEL SECURITY', tbl);
        EXECUTE format('ALTER TABLE workforce.%I FORCE ROW LEVEL SECURITY', tbl);
    END LOOP;
END $$;

CREATE POLICY org_isolation_select ON workforce.employees
    FOR SELECT USING (organization_id = current_setting('app.organization_id', true)::uuid);

CREATE POLICY org_isolation_insert ON workforce.employees
    FOR INSERT WITH CHECK (organization_id = current_setting('app.organization_id', true)::uuid);

CREATE POLICY org_isolation_update ON workforce.employees
    FOR UPDATE
    USING (organization_id = current_setting('app.organization_id', true)::uuid)
    WITH CHECK (organization_id = current_setting('app.organization_id', true)::uuid);

CREATE POLICY org_isolation_delete ON workforce.employees
    FOR DELETE USING (organization_id = current_setting('app.organization_id', true)::uuid);
```

**PII field-level access:** `date_of_birth`, `personal_email`, `emergency_contact` require `employees:read_sensitive` permission (enforced at application layer; optional column-level encryption for Enterprise).

## Soft Delete Strategy

| Entity | Soft Delete | Notes |
|--------|-------------|-------|
| Employees | Yes | Terminated employees retain record; `employment_status = 'terminated'` |
| Departments | Yes | Blocked if active employees assigned |
| Positions | Yes | Blocked if `filled_count > 0` |
| Employment Contracts | Yes | Active contracts cannot be soft-deleted; terminate first |
| Time Off Policies | Yes | Historical requests retain policy reference |
| Time Off Requests | Yes | Approved requests cannot be deleted; cancel instead |
| Payroll Runs | Yes | Completed runs immutable |
| Org Chart Nodes | Yes | Deactivate (`is_active = false`) preferred over delete |

## Audit Strategy

| Mechanism | Scope |
|-----------|-------|
| HR audit log | Employment status changes, salary changes, contract modifications |
| Temporal validity | `org_chart_nodes.effective_from/to`, contract date ranges |
| GDPR | Right-to-erasure cascades PII fields; retains anonymized employment record |
| Domain events | `workforce.employee.hired.v1`, `workforce.employee.terminated.v1`, `workforce.time_off.approved.v1` |
| Payroll integration | `provider_payload` stores external API response; never stores full SSN/bank details |
