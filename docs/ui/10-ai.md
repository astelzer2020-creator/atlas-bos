# AI Module — UI Specification

## Purpose

Complete UI specification for the **AI** module — the business brain of Atlas. Covers command center, persistent conversation panel, agent run history, configuration, memory browser, human-in-the-loop approval queue, budgets/models settings, prompt library, and all core interactions (ask, approve, reject, edit plan). Aligns with ARCH-17 and `ai_agents` schema (DB-12).

## Screen ID Convention

```
UI-AI-{NNN}       Screen / panel
UI-AI-MOD-{NNN}   Modal / dialog
UI-AI-INT-{NNN}   Interaction flow (cross-cutting)
```

## Module Navigation

| Route | Screen ID | Label |
|-------|-----------|-------|
| `/ai` | UI-AI-001 | AI Command Center |
| `/ai/runs` | UI-AI-003 | Run History |

**Global:** UI-AI-002 Conversation Panel available on all authenticated routes (lg+); collapses to FAB on xs–md.

## Permissions Matrix

| Permission | UI Effect |
|------------|-----------|
| `ai:agents:execute` | Ask questions, start agent runs, use conversation panel |
| `ai:agents:read` | View run history, read-only agent configs |
| `ai:agents:manage` | Create/edit agent definitions, tools, risk policies |
| `ai:approvals:read` | View approval queue |
| `ai:approvals:resolve` | Approve/reject/edit plans in queue |
| `ai:memory:read` | Browse memory entries |
| `ai:memory:write` | Edit/delete memory entries |
| `ai:memory:manage` | Memory retention, ingestion settings |
| `ai:settings:read` | View budgets and model config |
| `ai:settings:manage` | Set org budgets, model allowlist, default agents |
| `ai:prompts:read` | View prompt library |
| `ai:prompts:write` | Create/edit prompts |

**Effective agent permissions** = invoker perms ∩ profile tools ∩ tenant AI policy (BR-AGT-01).

High/critical risk tools show approval gate before execution (BR-AGT-02).

### Message Types

|------|-----------|

| Plan proposal | Numbered steps + action buttons (UI-AI-INT-003/004/005) |

| Control | Behavior |
|---------|----------|
| `[−]` | Collapse to 48px rail with unread badge |
| `[×]` | Close (session persists; reopen restores scroll) |
| Agent selector | Default from context; override per message |
| New conversation | `+` in header; archives current thread |

| Breakpoint | Behavior |

| xs–sm | Route `/ai/chat` full-screen (UI-AI-009) |
| md | 100% width overlay; swipe down to dismiss |

## Interaction Flows

### UI-AI-INT-001 — Ask Question

| Step | Actor | UI |
|------|-------|-----|
| 1 | User | Types in UI-AI-002 input or UI-AI-001 hero |
| 2 | System | Shows typing indicator; attaches page context |
| 3 | Planner/Analyst | Streams response or delegates to sub-agent |
| 4 | User | May follow up, click suggested actions, or open entity links |

**Keyboard:** `Cmd/Ctrl + K` opens panel focus; `Cmd/Ctrl + Enter` sends.

**Empty context:** Agent asks clarifying question rather than hallucinating entity IDs.

---

### UI-AI-INT-002 — Approve Action

| Step | Actor | UI |
|------|-------|-----|
| 1 | Executor | Posts plan card in conversation or approval queue item |
| 2 | User | Reviews steps, optional `View diff` for data mutations |
| 3 | User | Clicks **Approve** |
| 4 | System | Confirmation toast; run status → executing; audit log entry |
| 5 | Executor | Executes tools sequentially; progress updates in thread |

**Approve button:** Primary green; requires `ai:approvals:resolve`. Double confirmation for bulk > 100 records.

---

### UI-AI-INT-003 — Reject Action

| Step | Actor | UI |
|------|-------|-----|
| 1 | User | Clicks **Reject** on plan or queue item |
| 2 | System | UI-AI-MOD-001 Reject dialog opens |
| 3 | User | Selects reason + optional comment (required min 10 chars) |
| 4 | System | Run step cancelled; agent receives rejection context; may replan |

---

### UI-AI-INT-004 — Edit Plan

| Step | Actor | UI |
|------|-------|-----|
| 1 | User | Clicks **Edit plan** |
| 2 | System | UI-AI-MOD-002 Plan editor opens |
| 3 | User | Reorder, remove, or modify step parameters |
| 4 | User | **Submit revised plan** |
| 5 | System | Re-enters approval if risk elevated; else continues |

Editable fields per step type: entity IDs, email body, amounts (validated against user permissions).

## Cross-Cutting Requirements

### Accessibility

- Streaming text: `aria-live="polite"` region.
- Approval buttons: focus trap in modals; ESC cancels with confirm.
- Tool call cards: expandable via keyboard Enter/Space.

### Security UX

- Never display full API keys or PII in traces (masked).
- Show effective permission scope on each plan card.

### Observability

- Run cost displayed per message footer (small text).
- Link every action to audit trail entry.

### Localization

- Token/cost display respects org currency for budget UI.
- RTL: message bubbles mirror alignment.
