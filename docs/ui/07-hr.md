# HR UI Specification

## Purpose

Complete UI specification for the **Workforce** (HR) module: employee directory, org structure, time off, and employment contracts.

## Route Prefix

```
/app/:orgSlug/:workspaceSlug/hr/{resource}/:id?/{action}?
```

---

## Navigation Structure

```
HR
├── People
│   ├── Employee Directory        → /hr/employees
│   └── Employee Profile          → /hr/employees/:id
├── Organization
│   ├── Org Chart                 → /hr/org-chart
│   ├── Departments               → /hr/departments
│   └── Positions                 → /hr/positions
├── Time Off
│   ├── Calendar                  → /hr/time-off/calendar
│   └── Requests                  → /hr/time-off/requests
└── Contracts                     → /hr/contracts
    └── Detail                    → /hr/contracts/:id
```

Status flow: `pending → approved | rejected | cancelled`

#### Actions & Permissions

| Action | Permission | Actor |
|--------|------------|-------|
| View own requests | `hr:time_off:read` | Employee |
| View team requests | `hr:time_off:read_team` | Manager |

| Cancel others | `hr:time_off:admin` | HR |

#### Approval Workflow UI

1. Manager sees queue badge in nav
2. Approve → optional comment; balance deducted
3. Deny → **required** reason (shown to employee)
4. Insufficient balance → inline error on submit

#### Responsive

- **Mobile**: Card per request; swipe right approve / left deny (manager only)
- **Tablet**: Split: list + detail panel

## Permission Matrix Summary

| Role | Typical Permissions |
|------|---------------------|
| **Employee (self)** | `hr:employees:read` (own), `hr:time_off:read/write` (own), `hr:time_off:write` |
| **Manager** | + `hr:employees:read_team`, `hr:time_off:read_team`, `hr:time_off:approve`, `hr:org_chart:read` |
| **HR Admin** | + `hr:employees:*`, `hr:contracts:*`, `hr:compensation:read`, `hr:time_off:read_all` |
| **Viewer** | `hr:employees:read`, `hr:org_chart:read`, `hr:departments:read` |

---

## ABAC Rules (UI Enforcement)

| Rule | UI Behavior |
|------|-------------|
| Compensation sensitivity | Blur/mask; reveal on click with audit log |
| Cross-department view | Managers see only subtree unless `read_all` |
| Contractor vs FTE | Different contract fields shown |
| GDPR data export | Self-service from profile → Privacy section |

---

## Accessibility

- Org chart: keyboard navigation between nodes (Tab/Arrow); list mode fallback
- Calendar: `aria-label` on each event with employee, type, dates
- Approval actions: not swipe-only; always available as buttons
- Color-blind safe leave type legend (patterns + labels)

---

## Localization

- Date ranges respect locale (Mon–Fri vs Sun–Thu work weeks configurable)
- Tenure display localized
- Contract types translatable
